## 1. 产品概述

| 项目     | 内容                                                      |
| -------- | --------------------------------------------------------- |
| 产品名称 | 问玄东方·寺院管理台                                        |
| 产品定位 | 面向寺院管理员的Web端后台管理工具，用于管理寺院信息、法师、服务及预约订单 |
| 使用对象 | 寺院管理员（由寺院指定的管理运营人员）                      |
| 端类型   | Web端（PC浏览器，推荐Chrome/Edge最新版）                   |
| 登录方式 | 手机号 + 验证码 / 账号 + 密码                              |
| 数据范围 | 仅显示当前登录管理员所属寺院的数据，数据隔离               |

### 1.1 设计风格

管理台采用专业Web后台风格，与C端App的"红金火土"暗色主题保持品牌一致性，同时在后台场景下优化为更舒适的阅读体验。

| 用途           | 色值                        | 说明                       |
| -------------- | --------------------------- | -------------------------- |
| 侧边栏背景     | `#1C1210`                   | 深檀木色，与C端主背景一致  |
| 内容区背景     | `#F5F0EB`                   | 暖白色，后台阅读舒适       |
| 主操作色       | `#C45A3C`                   | 朱砂红，主CTA按钮          |
| 强调色         | `#C8A96E`                   | 暗金色，图标/高亮           |
| 侧边栏文字     | `#C5B097`                   | 默认灰金                   |
| 侧边栏激活     | `#F0E6DA`                   | 激活文字色                 |
| 内容区主文字   | `#2D2420`                   | 深棕黑，阅读主色           |
| 内容区次文字   | `#7A6B5E`                   | 中灰棕，辅助说明           |
| 卡片背景       | `#FFFFFF`                   | 白色卡片                   |
| 边框           | `#E8E0D8`                   | 浅暖灰边框                 |
| 成功状态       | `#5B8C5A`                   | 佛教绿                     |

### 3.1 登录页

| 属性     | 值                |
| -------- | ----------------- |
| 页面路径 | `/login`          |
| 页面描述 | 寺院管理员登录入口，支持手机号验证码和账号密码两种登录方式 |

#### 页面布局

页面分为左右两部分（或移动端上下排列）：

```
┌──────────────────────────────────────────────┐
│                                              │
│   ┌────────────────┐  ┌──────────────────┐   │
│   │                │  │  问玄东方·寺院管理台│  │
│   │   品牌插图区    │  │                  │   │
│   │  寺院/法器      │  │  [登录方式Tab切换] │   │
│   │  相关背景图     │  │  ┌──────────────┐ │   │
│   │                │  │  │ 手机号/账号   │ │   │
│   │   (背景色      │  │  ├──────────────┤ │   │
│   │    #1C1210)    │  │  │ 验证码/密码   │ │   │
│   │                │  │  ├──────────────┤ │   │
│   │                │  │  │ [获取验证码]  │ │   │
│   │                │  │  ├──────────────┤ │   │
│   │                │  │  │ ☐ 记住登录   │ │   │
│   │                │  │  ├──────────────┤ │   │
│   │                │  │  │   登 录       │ │   │
│   │                │  │  └──────────────┘ │   │
│   └────────────────┘  └──────────────────┘   │
│                                              │
└──────────────────────────────────────────────┘
```

#### 数据字段

| 字段名     | 类型     | 必填 | 描述                                     |
| ---------- | -------- | ---- | ---------------------------------------- |
| 登录方式   | 枚举     | 是   | `phone_code`（手机号+验证码）/ `password`（账号+密码） |
| 手机号     | String   | 是   | 11位手机号码，手机号验证码登录时必填       |
| 账号       | String   | 是   | 管理员账号，密码登录时必填                  |
| 密码       | String   | 是   | 6-20位密码，需包含字母和数字               |
| 验证码     | String   | 是   | 6位数字验证码，60秒倒计时重发               |
| 记住登录   | Boolean  | 否   | 勾选后7天内免登录（Token续期）              |

#### 交互说明

1. **登录方式切换**：顶部Tab切换「验证码登录」/「密码登录」，切换时表单字段联动切换，带有平滑过渡动画
2. **获取验证码**：点击后手机号校验通过则发送验证码，按钮进入60秒倒计时，显示"重新获取(59s)"
3. **记住登录**：勾选后使用长效Refresh Token，有效期为7天；未勾选则Token有效期为当前浏览器会话
4. **登录成功**：跳转至仪表盘 `/dashboard`
5. **登录失败**：表单内联红色提示文字（如"手机号格式错误"、"验证码已过期"）
6. **品牌插图区**：左侧展示寺院品牌相关装饰性插图，背景色为 `#1C1210`，右侧为登录表单区，背景色为 `#F5F0EB`

#### 业务规则

| 规则编号 | 规则描述                                                                 |
| -------- | ------------------------------------------------------------------------ |
| BR-001   | 手机号必须为11位中国大陆手机号，以1开头                                   |
| BR-002   | 验证码有效期5分钟，同一手机号60秒内不可重复发送                           |
| BR-003   | 密码需包含至少一个字母和一个数字，长度6-20位                              |
| BR-004   | 连续5次登录失败则锁定账号30分钟，锁定期间不可登录                         |
| BR-005   | 管理员账号需由平台后台预先创建，管理员无法自行注册                         |
| BR-006   | 登录成功后Token存储于HttpOnly Cookie，防止XSS攻击                          |

### 3.2 仪表盘 (Dashboard)

| 属性     | 值                         |
| -------- | -------------------------- |
| 页面路径 | `/dashboard`               |
| 页面描述 | 管理台首页，汇总展示今日运营数据和待办事项，帮助管理员快速掌握寺院运营状况 |

#### 页面布局

```
┌─────────────────────────────────────────────────┐
│ 侧边栏 │ 内容区                                   │
│        │  ┌─────────────────────────────────────┐  │
│  仪表盘 │  │  今日数据 (4张统计卡片横向排列)       │  │
│  寺院   │  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐│  │
│  法师   │  │  │今日预约│ │待处理 │ │今日收入│ │平均评分││  │
│  服务   │  │  └──────┘ └──────┘ └──────┘ └──────┘│  │
│  预约   │  └─────────────────────────────────────┘  │
│  评价   │  ┌──────────────────┐ ┌───────────────┐  │
│  报表   │  │ 近7天预约趋势图   │ │  待办事项提醒  │  │
│  设置   │  │ (折线图)         │ │  · 待确认预约  │  │
│        │  │                  │ │  · 待回复评价  │  │
│        │  └──────────────────┘ └───────────────┘  │
│        │  ┌─────────────────────────────────────┐  │
│        │  │ 最新预约 (最近5条表格)                 │  │
│        │  │ 订单号 | 用户 | 服务 | 日期 | 状态   │  │
│        │  │  ...  |  ... |  ... |  ... |  ...   │  │
│        │  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

#### 数据字段

**今日数据卡片**

| 字段名     | 类型   | 必填 | 描述                                           |
| ---------- | ------ | ---- | ---------------------------------------------- |
| 今日预约数 | Number | 是   | 今日产生的预约订单总数，含所有状态               |
| 待处理预约 | Number | 是   | 状态为"待确认"的预约订单数                      |
| 今日收入   | Number | 是   | 今日已完成订单的总金额（服务费+功德金），单位元   |
| 平均评分   | Number | 是   | 近30天所有评价的加权平均分，保留一位小数          |

**预约趋势图**

| 字段名 | 类型     | 必填 | 描述                          |
| ------ | -------- | ---- | ----------------------------- |
| 日期   | String   | 是   | 近7天日期，格式 MM-DD         |
| 预约数 | Number   | 是   | 当日预约总量                  |
| 确认数 | Number   | 是   | 当日已确认预约数              |
| 取消数 | Number   | 是   | 当日取消预约数                |

**最新预约列表**

| 字段名     | 类型   | 必填 | 描述                              |
| ---------- | ------ | ---- | --------------------------------- |
| 订单号     | String | 是   | 预约订单唯一编号，如 BK20260629001 |
| 用户昵称   | String | 是   | 下单用户的昵称                    |
| 服务类型   | String | 是   | 预约的服务类型名称                |
| 预约日期   | String | 是   | 预约日期，格式 YYYY-MM-DD         |
| 状态       | String | 是   | 订单当前状态（待确认/已确认/已完成/已取消） |
| 用户头像   | String | 是   | 用户头像URL缩略图，40x40px        |

**待办事项提醒**

| 字段名     | 类型   | 必填 | 描述                              |
| ---------- | ------ | ---- | --------------------------------- |
| 待办类型   | String | 是   | `pending_booking`（待确认预约）/ `pending_reply`（待回复评价） |
| 数量       | Number | 是   | 该类型待办数量                    |
| 跳转链接   | String | 是   | 点击后跳转的页面路径              |

#### 交互说明

1. **数据卡片**：4张卡片横向排列，每张卡片包含图标 + 数值 + 标签。数值使用大号字体(28px)，标签使用14px次文字色。卡片悬停时轻微上浮（translateY -2px）+ 阴影加深
2. **趋势图**：采用折线图，X轴为近7天日期，Y轴为预约数量。双折线展示"预约总量"和"已完成"两条曲线，区域填充半透明色。图表交互：鼠标悬浮显示当日详细数据Tooltip
3. **待办事项**：卡片式列表，每条待办显示类型图标 + 文字描述 + 数量Badge。点击整条待办跳转至对应管理页面并自动应用筛选条件
4. **最新预约**：表格展示最近5条预约，状态列使用彩色标签（待确认-黄色/已确认-蓝色/进行中-绿色/已完成-灰色/已取消-红色）。点击行可跳转至预约详情
5. **自动刷新**：仪表盘数据每60秒自动刷新，不刷新页面

#### 业务规则

| 规则编号 | 规则描述                                           |
| -------- | -------------------------------------------------- |
| BR-010   | 今日数据统计时区与寺院所在地一致                     |
| BR-011   | 趋势图展示近7天（含今天）的数据                      |
| BR-012   | 待办事项按紧急程度排序：待确认预约 > 待回复评价       |
| BR-013   | 最新预约列表仅展示最近5条，按创建时间倒序             |
| BR-014   | 今日收入统计包含服务费和功德金两部分                  |

#### 交互说明

1. **表单编辑**：所有字段均可编辑，未修改时表单控件为展示模式；点击"编辑"按钮后进入编辑模式
2. **图片管理**：
   - 点击"上传图片"触发文件选择器，支持批量上传，单张不超过5MB
   - 图片上传后自动进入预览网格，显示为缩略图（120x90px）
   - 图片支持拖拽排序（HTML5 Drag & Drop）
   - 每张图片右上角有删除按钮（X）和设为封面按钮（星标）
   - 封面图片左上角显示"封面"角标
3. **地图标注**：嵌入第三方地图组件，可通过搜索地址自动定位，也可手动拖拽标记点精确标注

## 6. 寺院数据边界与法师管理规则

寺院管理台不是全平台主数据入口，只能维护当前登录账号绑定的寺院。登录成功后，后端以 JWT 中的 `templeId` 作为数据权限边界，前端传参不得切换寺院。

### 6.1 可管理范围

| 模块 | 范围 |
| ---- | ---- |
| 寺院信息 | 仅当前绑定寺院 |
| 寺院服务 | 仅当前绑定寺院的服务，服务码来自统一服务字典 |
| 法师列表 | 仅 `master.temple_code=当前寺院` 的法师 |
| 预约订单 | 仅当前寺院订单 |
| 加持任务 | 仅派发到当前寺院的任务 |
| 评价/报表 | 仅当前寺院数据 |

### 6.2 法师创建与上架

| 步骤 | 操作 | 结果 |
| ---- | ---- | ---- |
| 1 | 寺院管理员创建法师 | 后端自动写入当前 `templeId`，不允许创建无寺院法师 |
| 2 | 平台审核法师资质 | 审核通过后法师可进入上架流程 |
| 3 | 平台分配大师 App 账号 | 法师账号绑定 `master_id`，并派生所属寺院 |
| 4 | 法师上架 | C端找师傅和寺院详情页才展示该法师 |

### 6.3 服务标签规则

找寺院左侧服务筛选、寺院卡片服务标签、预约服务入口均来自 `temple_service`。寺院管理台新增服务时只能选择统一服务字典中的服务码，不能自定义另一套筛选标签。
