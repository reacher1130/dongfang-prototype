# 产品原型交互完整性审计报告

> 审计日期：2026-07-18
> 审计对象：C 端、法师端、寺院管理台、商城管理台、平台管理台及总入口
> 事实源：GitHub Pages 实际发布的 HTML 与相对链接；`.design` 仅保留为历史设计元数据

## 1. 结论

当前原型共有 108 个 HTML 页面，扫描到 1471 个站内相对链接，缺失目标为 0。五端都可从各自登录页或首页进入主导航，并可抵达新增闭环页。

这只证明“页面和导航完整”，不等同于生产功能已经接通。支付页面明确标注本地模拟；直播默认关闭；平台备份只展示当前已实现的手动全量快照；外部模型、云直播、真实支付和代付不在原型中伪装为已上线。

## 3. 六项改进对应

| 需求 | 原型落点 | 交互状态 |
|------|----------|----------|
| AI 问事 | `ai-divination.html` | 直接对话、历史会话、新建会话和失败重试状态齐全 |
| 大师广场 | `community.html`、法师 `media-studio.html`、平台内容审核页 | 图文/视频浏览、发布与审核入口齐全 |
| 设计广场下单 | `diy-detail.html` → `diy-order.html` → `payment-result.html` | 地址、动态加持服务、服务端最终计价和模拟支付说明齐全 |
| 视频与直播 | C 端社区/视频、法师媒体工作室 | 媒体上传/播放可见；直播关闭态明确，不展示假推流成功 |
| 信仰流派 | `belief-topic.html`、寺院/大师列表 | 一级流派专题及筛选入口齐全 |
| 诉求聚合 | `intention-hub.html` | 商品与寺院服务混排，分别进入真实下单落点 |

## 4. 补齐页面

- C 端商城：购物车、商品详情、确认订单、支付结果、统一订单列表。
- C 端个人中心：订单分类、资料/地址/评价/优惠券等二级入口。
- C 端大师广场：图文与视频内容入口。
- 法师端：媒体工作室和请假申请。
- 三个管理台：忘记密码或受控的管理操作入口。
- 平台备份：手动全量、下载和文件名确认恢复；自动/增量明确未启用。

## 5. 仍需代码闭环的原型能力

| 编号 | 风险 | 原型表现 | 当前代码事实 |
|------|------|----------|--------------|
| P1 | 高 | 生产登录/支付可配置 | 短信仍为固定开发验证码、密码仍为明文兼容；真实微信/支付宝未接入 |
| P2 | 中 | 寺院/商城报表有趋势和排行 | 后端部分图表仍构造样例趋势/排行，不能作为经营数据 |
| P3 | 中 | 云 AI/直播可以启用 | 需要外部模型、VOD/直播账号、密钥和回调域名后另行验收 |
| P4 | 中 | 物流/提现可以推进 | 本地 Provider 会模拟轨迹和打款，生产 Provider 尚未接入 |
