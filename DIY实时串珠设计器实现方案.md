# DIY 实时串珠设计器实现方案

> 方案状态：首版已实现。交互原型见 `问玄东方App/pages/diy-designer-demo.html`；原 `diy-design.html` 兼容入口及 DIY 广场、详情、订单返回入口均已指向该页面。C 端 iOS 已使用同一 v2 文档和交互模型。

## 1. 目标与产品边界

目标是把当前“材料列表 + 数量购物车”升级为可直接操作的手串设计器：用户选择珠子后实时看到成品，能旋转手串、拖动换位、拖出删除、撤销/重做，并根据手围获得松紧提示和预估价格。保存后再次打开必须一比一恢复，订单仍由服务端按当前 SKU、状态和库存重新计价。

首版采用原生 2.5D 圆环渲染，不依赖 WebView。它能覆盖核心体验、适配更多 iPhone 型号，并允许沿用普通图片素材。真实光照、自由视角和 AR 试戴列为后续增强，不阻塞首版交易闭环。

首版不实现珠子碰撞模拟、绳结物理和照片级材质扫描；这些投入高，但对“选珠、调序、下单”的验证价值有限。

## 2. 当前基础与差距

当前 C 端 `DiyDesignView` 已升级为实时圆环编辑器，`DiyViewModel` 同时维护有序珠位和聚合订单项，保存的 `designData` 可精确恢复珠位、手围与绳线。页面已支持旋转、换位、拖出删除、撤销/重做、本机草稿、松紧提示，并继续串联现有保存、地址、加持、服务端计价和支付页面。

后端已有真实材料库、SKU 库、设计保存、服务端重新计价、库存扣减、不可变设计/计价快照、支付和加持订单闭环。现有 `design_data` 是 JSON 文本，可兼容新格式；下单解析器目前识别 `items` 或 `materials`，因此新格式必须同时保存有序 `beads` 和聚合 `items`。

## 3. 建议交互

1. 用户先选择手围，系统显示建议用珠长度。
2. 点击材料卡，珠子以飞入动画加入圆环并自动选中。
3. 在圆环空白处横向滑动可旋转；拖动珠子到另一角度可换位。
4. 将珠子拖出圆环进入移除区即可删除；选中后也可复制或删除。
5. 每次结构变化进入撤销/重做栈，自动保存本地草稿。
6. 顶部持续显示松紧状态和预估金额；库存不足、材料下架、价格变化在确认订单时给出明确差异。
7. “完成设计”先保存有序设计，再进入地址、加持服务、服务端计价和支付流程。

Demo 中已覆盖以上主要手势、动效和状态反馈。视觉采用炭黑工作台、玉绿色状态、朱红主操作和金色价格，突出材质本身，不把设计器做成商城卡片堆叠页。

## 4. 客户端技术设计

### 4.1 模块拆分

- `BraceletDesignerView`：页面编排、材料面板、底部操作和无障碍语义。
- `DiyBraceletStage`：SwiftUI 椭圆布局和手势层；每颗珠子独立命中，`Canvas` 只绘制可复现的材质纹理。
- `BraceletLayoutEngine`：输入手围、珠径、有序珠位和旋转角，输出每颗珠子的中心、缩放、层级与命中区域。
- `BraceletGestureController`：统一处理旋转、选中、拖动换位、拖出删除，避免手势互相抢占。
- `DiyDesignDocument`：可编码的版本化设计文档，是界面状态、草稿和后端请求的唯一数据源。
- `DesignHistory`：最多保留 50 个结构快照；旋转视角不进入历史，新增、删除、换位和手围变化进入历史。
- `BeadAssetCache`：用 `NSCache` 缓存缩略图和 2x/3x 渲染图，失败时使用服务端下发的主色生成材质球占位。

### 4.2 排列与动效

圆环采用椭圆投影。第 `i` 颗珠子的角度为 `rotation + 2πi/n`，位置由椭圆参数计算；下半环提高缩放和 zIndex，形成前后景深。珠径参与实际显示尺寸，隔片可使用独立扁圆轮廓。

拖动时把触点转换为圆环中心角，映射到最近珠位；手指松开后用弹簧动画吸附。拖出阈值建议为标准化半径 `1.35`，进入阈值时给予轻触觉反馈。添加动画从材料卡中心飞向目标珠位，结构更新必须在动画开始前完成，保证快速连续点击不丢数据。

松紧计算以毫米为统一单位：`usedLength = Σ 珠子有效直径 + 隔片间隙 + 固定配件长度`，目标长度为 `手围 + 佩戴余量`。余量阈值由商品配置维护，界面只显示“可继续加入 / 合适 / 偏紧”，服务端保存时再次校验。

### 4.3 设计文档 v2

```json
{
  "version": 2,
  "wristSizeMm": 160,
  "fitAllowanceMm": 5,
  "cord": { "materialId": 14, "skuId": 0, "materialName": "弹力绳", "unitPrice": 2 },
  "beads": [
    {
      "slotId": "4C28...",
      "position": 0,
      "materialId": 2,
      "skuId": 0,
      "diameterMm": 10,
      "materialName": "星月菩提",
      "spec": "10mm",
      "unitPrice": 18,
      "subtype": "main_bead",
      "image": "/assets/materials/bodhi.jpg"
    }
  ],
  "items": [
    { "materialId": 2, "skuId": 0, "materialName": "星月菩提", "spec": "10mm", "quantity": 12, "unitPrice": 18 },
    { "materialId": 14, "skuId": 0, "materialName": "弹力绳", "quantity": 1, "unitPrice": 2, "subtype": "cord" }
  ]
}
```

`beads` 是显示与恢复的权威顺序；`items` 是兼容现有下单解析器的聚合索引。客户端的 `unitPrice` 仅用于展示，服务端下单继续忽略它并按 SKU 重新查询。旧版数组格式读取后转换为 v2，无法获知原顺序时按原材料顺序展开，并提示用户重新确认排列。

## 5. 后续后端与素材增强

以下内容是增强阶段设计，不属于当前已实现契约。当前后端已经持久化完整 v2 JSON，并从 `items` 聚合投影完成服务端重新计价；尚未提供 quote、草稿修订和独立渲染素材 API。

### 5.1 材料渲染数据

`material` 或独立 `material_render_asset` 增加以下可运营字段：

- `thumbnail_url`：材料面板图片。
- `texture_url`：透明背景标准珠素材，至少 512x512。
- `shape`：`round/spacer/three_way/pendant/tassel/cord`。
- `diameter_mm`、`width_mm`：排布与尺寸计算参数。
- `dominant_color`、`highlight_strength`、`roughness`：素材缺失时的降级渲染参数。
- `asset_version`、`status`：缓存失效与素材审核状态。

商城管理台补充渲染素材上传、裁切预览和版本发布。素材变更不能覆盖订单快照使用的历史版本。

### 5.2 后续 API 建议

- `GET /api/v1/diy/materials`：响应扩展 `skuId/diameterMm/shape/renderAsset`，旧字段保留。
- `POST /api/v1/diy/designs/quote`：提交 v2 文档，返回权威材料状态、预估价格、松紧结果和逐项差异，不扣库存。
- `POST /api/v1/diy/designs`：服务端校验文档版本、珠位连续性、`beads` 与 `items` 数量一致，并生成规范化文档和预估总价。
- `PUT /api/v1/diy/designs/:id`：保存草稿修订，使用 `revision` 做乐观锁，避免多设备覆盖。
- `POST /api/v1/diy/designs/:id/order`：保持现有路径与事务计价；订单快照保存完整 v2 文档和素材版本。

保存设计不锁库存；进入下单事务后才按聚合 `items` 锁定并扣减库存。材料下架或库存不足时，响应必须指出受影响的 `slotId/materialId`，客户端高亮对应珠子。设计展示价由 quote/save 的服务端结果更新，不能继续信任客户端 `totalPrice`。

### 5.3 安全与兼容

- JWT 用户身份覆盖请求中的 `userId`。
- 限制最多珠位、JSON 大小、单材质数量和支持的文档版本。
- 服务端重新聚合 `beads` 并与 `items` 对照，防止只在兼容字段里少报数量。
- v1 设计保持可查询和下单；编辑时迁移到 v2，不批量改写历史订单快照。
- 设计广场缩略图由服务端异步生成并关联设计修订，避免每个列表单元现场渲染完整圆环。

## 6. 分阶段落地

### 阶段 A：2.5D 编辑器（已完成）

已完成 v2 客户端模型、椭圆布局、程序化材质、手势、历史栈、本地草稿、v1/v2 恢复及现有下单页串联。后端兼容读取 v1/v2，并在创建订单时重新查询价格、状态和库存；设计保存接口当前只保存客户端预估，不生成服务端 quote。

### 阶段 B：真实素材与运营能力

建立标准珠素材规范，商城管理台增加渲染素材、物理尺寸和版本管理；设计广场增加服务端缩略图。补齐价格变化、下架、缺货珠位定位和替代材料推荐。

### 阶段 C：3D 与 AR 增强

在业务数据稳定后，以 RealityKit/Metal 材质替换 2.5D 渲染层，复用同一 `DiyDesignDocument` 和手势状态机。增加自由视角、真实光照与手腕 AR 试戴，不改变保存和下单契约。

## 7. 验收标准

- 30 颗珠子在目标 iPhone 上旋转和拖动保持流畅，常规操作反馈不超过 100 ms。
- 连续快速加入、跨位拖动、拖出删除、撤销和重做均不丢珠、不重复珠位。
- 杀进程、重新登录和跨设备拉取后，珠位顺序、手围、配件与素材版本可精确恢复。
- v1 旧设计仍能查看和下单；迁移为 v2 后能够再次编辑。
- 篡改客户端价格、数量或聚合项不能绕过服务端重新聚合、计价和库存校验。
- 下单价格变化、材料下架和库存不足可定位到圆环中的具体珠子。
- VoiceOver 能按顺序读出材质与珠位，开启“减弱动态效果”后使用淡入和吸附替代飞行动画。
- iPhone 小屏、标准屏和大屏均无控件遮挡，底部操作区适配安全区。

## 8. 已采用的首版决策

1. 首版采用原生 2.5D，真 3D/AR 放到增强阶段。
2. 手围提供 14-20 cm、每 1 cm 一个固定档位。
3. 设计广场沿用现有审核和上下架状态，不开放无审核发布。
4. 首版使用程序化材质降级展示；标准素材上传与版本管理留到阶段 B。
5. 珠子及配件使用统一有序珠位；绳线作为独立配置，不占圆环珠位。
